# Интеллектуальная музыкальная система «TuneFlow» (Курсовой проект)

Данный программный комплекс представляет собой высокотехнологичное решение, реализованное в рамках курсового проекта. Программа предназначена для управления фонотекой из 1000 музыкальных треков и предоставляет персонализированные рекомендации на основе современных алгоритмов анализа графов и коллаборативной фильтрации.

## Функциональные возможности

Комплекс разделен на три ключевых модуля анализа и управления:

### 1. Алгоритмическое ядро и обработка данных (Проект Logic)
*   **Сортировка данных (QuickSort):** Реализация авторского алгоритма быстрой сортировки с поддержкой динамического выбора критерия (рейтинг, год, название) и поддержкой реверсивного порядка. Временная сложность: $O(n \log n)$.
*   **Многопараметрический поиск:** Система фильтрации, позволяющая выполнять выборку данных по совокупности признаков (жанр, исполнитель, временной диапазон) в реальном времени.
*   **Перманентность данных:** Использование формата JSON для хранения базы из 1000 записей, что обеспечивает неизменность данных между сессиями работы приложения.

### 2. Интеллектуальные рекомендательные алгоритмы
*   **Графовая модель (User-Track Graph):** Построение двудольного графа связей между слушателями и композициями на основе списков смежности.
*   **Алгоритм BFS (Breadth-First Search):** Реализация поиска в ширину по графу предпочтений для генерации рекомендаций «через рукопожатие» (социальный поиск).
*   **Коллаборативная фильтрация:** Использование коэффициента сходства Жаккара для сопоставления профилей пользователей и выявления релевантного контента на основе «музыкального соседства».
*   **Content-based Similarity:** Математический расчет похожести треков на основе взвешенных коэффициентов (Жанр — 70%, Эпоха — 30%) для поиска стилистически близких произведений.

### 3. Инфраструктура интерфейса и визуализация (Проект UI)
*   **Modern Dark UI:** Графический интерфейс в стиле актуальных стриминговых сервисов (Spotify Style) с акцентом на эргономику и читаемость данных.
*   **Динамическая визуализация:** Модуль статистического анализа, строящий графики распределения фонотеки по годам на базе компонента `Canvas`. Масштабирование графиков выполняется программно с учетом размеров окна.
*   **Система плейлистов:** Механизм визуального выделения (подсветка строк) и управления персональными коллекциями пользователя.

## Технологический стек
*   **Язык программирования:** C# 12.0 (.NET 8.0)
*   **Интерфейс:** WPF (Windows Presentation Foundation) с использованием XAML-стилизации.
*   **Формат данных:** JSON (музыкальная база 1000 записей).
*   **Архитектура:** Layered Architecture (Разделение на Logic и UI слои).
*   **Инструментарий:** Visual Studio 2022.

## Исследование алгоритмической сложности
В рамках проекта проведена оптимизация ключевых вычислительных узлов:
1.  **Поиск в ширину (BFS):** Оптимизирован за счет использования множества (HashSet) посещенных узлов, что предотвращает зацикливание в графе и ограничивает сложность до $O(V + E)$.
2.  **Сортировка:** Применение ручной реализации QuickSort вместо стандартных методов позволило интегрировать поддержку кастомных компараторов для сложных объектов `Track`.
3.  **Визуализация:** Отрисовка графиков реализована через итеративный проход по статистическим выборкам, что минимизирует нагрузку на графическую подсистему при обновлении данных.

## Инструкция по запуску
1.  Клонируйте репозиторий: `git clone https://github.com/ваш_логин/TuneFlow.git`
2.  Откройте файл решения `TuneFlow.sln` в Visual Studio 2022.
3.  Выполните сборку решения (Build Solution).
4.  Программа автоматически сгенерирует файл базы данных `music_db.json` при первом запуске (если он отсутствует).
5.  Запустите проект `TuneFlow.UI`.
